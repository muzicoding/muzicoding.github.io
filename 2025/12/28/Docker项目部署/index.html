3. DockerCompose

部署一个简单的java项目，其中包含3个容器：

  • MySQL
  • Nginx
  • Java项目

稍微复杂的项目，其中还会有各种各样的其它中间件，需要部署的东西远不止3个。如果还像之前那样手动的逐一部署，就太麻烦了。

而Docker Compose就可以实现多个相互关联的Docker容器的快速部署。它允许用户通过一个单独的 docker-compose.yml 模板文件（YAML 格式）来定义一组相关联的应用容器。

3.1 基础语法

docker-compose.yml文件的基本语法可以参考官方文档：

https://docs.docker.com/compose/compose-file/compose-file-v3/

与docker run命令后的参数类似

docker run 参数 docker-compose 字段 说明
--name container_name 容器名称
-p ports 端口映射
-e environment 环境变量
-v volumes 数据卷配置
--network networks 网络

3.2 常见命令

编写好docker-compose.yml文件，就可以部署项目了。常见的命令：

https://docs.docker.com/reference/cli/docker/compose/

基本语法如下：

类型 参数或指令 说明
Options -f 指定 compose 文件的路径和名称（如 -f ./my-compose.yml）
-p 指定 project 名称。project 是当前 compose 文件中多个 service 的集合，是一个逻辑分组概念
Commands up 创建并启动所有 service 容器（加 -d 可后台运行）
down 停止并移除所有容器、网络、以及默认创建的卷（若未指定 --volumes 则不删卷）
ps 列出当前 project 中所有已启动的容器
logs 查看指定服务容器的日志（如 docker-compose logs web）
stop 停止正在运行的容器（不删除）
start 启动已停止的容器
restart 重启容器
top 查看指定服务中正在运行的进程
exec 在指定的运行中容器内执行命令（如 docker-compose exec db psql）
