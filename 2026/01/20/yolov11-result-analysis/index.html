YOLO训练结果解析

发表于2026-01-20|更新于2026-01-25|YOLOv11
|浏览量:

1. 训练过程中

练过程中会输出以下信息，包含训练指标和验证指标。

1.1 训练指标

  • Epoch 表示当前训练轮数
  • GPU_mem 表示当前训练使用的GPU显存，为0G表示使用CPU进行训练
  • box_loss 边界框回归损失（Bounding Box Loss）
    衡量预测框与真实框（ground truth）在位置和尺寸上的误差。值越小越好。通常基于IoU 。
  • cls_loss 分类损失（Classification Loss）
    衡量类别预测的准确性。常用交叉熵损失（Cross-Entropy）。值越小越好。
  • dfl_loss 分布焦点损失（Distribution Focal Loss）
    用于更精细地回归边界框坐标。它将框的坐标建模为离散分布，提升定位精度。值越小越好。
  • Instances 当前batch中包含的目标实例（objects）总数
  • img_size 当前训练的图像大小

1.2 验证指标

  • Class all表示所有类别的综合结果
  • Images 验证集中图像数量
  • Instances 验证集中总目标数量
  • Box(P) Precision精确率
    预测为正样本中，真正为正的比例。公式：TP / (TP + FP)
  • R Recall召回率
    所有真实正样本中，被正确检出的比例。公式：TP / (TP + FN)
  • mAP50 在IoU阈值为0.5时的平均精度（对所有类别取平均）
    越高越好（理想接近 1.0）。
  • mAP50-95 在多个 IoU 阈值（0.5, 0.55, …, 0.95）下的平均 mAP
    更严格、更全面的指标

2. 训练结果

训练完成后，会生成以下文件：

2.1 weights

best.pt：验证集上 mAP50 最高的模型权重（通常用于最终部署）。

last.pt：最后一个 epoch 的模型权重（可用于断点续训）。

2.2 args.yaml

保存了训练时的所有参数，包括：

  • 数据集路径（data: xxx.yaml）
  • 模型大小（model: yolov11s.pt）
  • 图像尺寸（imgsz: 640）
  • batch size、epochs、学习率等

可用于复现实验，修改后用于微调（fine-tune），查看是否用了预训练权重（pretrained: True）

2.3 results.csv和result.png

results.csv包含每个epoch的详细指标：

epoch, train/box_loss, train/cls_loss, val/box_loss, val/cls_loss
metrics/precision(B), metrics/recall(B), metrics/mAP50(B), metrics/mAP50-95(B)…

results.png是训练过程可视化图，通常包含 10 个子图：

Box Loss，Cls Loss，dfl Loss，Precision，Recall，mAP50 & mAP50-95

可用于快速判断训练是否正常，Loss 应持续下降，metrics 应持续上升

若验证指标震荡或下降 → 可能过拟合或学习率太高

2.4 labels.jpg

显示训练集中所有标注框的中心点和宽高分布（归一化到 [0,1]）。

用于检查标签是否合理（例如是否有异常大/小的框），分析目标在图像中的位置偏好（如是否集中在中心）

  • 左上角 类别柱状图，显示训练集中类别的实例数量。
  • 右上角 标注框分布图，显示训练集中标注框的分布。
  • 左下角 中心点热力图（x, y）
    • 坐标轴：x 和 y 是归一化后的中心点坐标（范围 [0,1]）
    • 颜色深浅：表示该位置出现目标框的频率（越深 = 越多）
    • （可观察到当前目标主要出现在图像中心）
  • 右下角 宽高比分布（width vs height）
    • width：归一化宽度（[0,1]）
    • height：归一化高度（[0,1]）
    • 分布呈斜线趋势 → 宽高比相对固定
    • （可观察到目标尺寸较为一致，可能是同一类型物体）

2.5 BoxF1_curve.png

F1 分数 vs 置信度阈值曲线

F1 = 2×(P×R)/(P+R)，综合指标

峰值对应的 conf 是最佳阈值

Y轴：F1-score（0～1）
X轴：Confidence Threshold（置信度阈值，0～1）

高 F1：表示模型在“准确率”和“召回率”之间取得了良好平衡。

F1=1.0：意味着 Precision=Recall=1.0 → 完美检测（无误检、无漏检）。

所有类别在 confidence = 0.858 时达到 F1=1.00 → 表示此时预测最优

2.6 BoxP_curve.png和BoxR_curve.png

Precision vs 置信度阈值曲线
峰值对应的 conf 是最佳阈值

置信度越高，Precision 通常越高（但 Recall 下降）

Recall vs 置信度阈值曲线
峰值对应的 conf 是最佳阈值

置信度越高，Recall 越低（因为过滤掉更多低分预测）

2.7 BoxPR_curve.png

Precision vs Recall 曲线

2.8 confusion_matrix.png

混淆矩阵

混淆矩阵用于评估模型性能，显示模型在各个类别之间的分类情况。

混淆矩阵结构：

  • 行：真实类别（Ground Truth）
  • 列：预测类别（Predicted）
  • 对角线：正确分类的数量（越亮越好）
  • 非对角线：错误分类的数量（越暗越好）

混淆矩阵的用途：

  • 评估模型性能：通过观察混淆矩阵，可以了解模型在各个类别之间的分类情况。
  • 识别问题：如果某些类别的混淆矩阵存在大量错误分类，则说明模型存在问题。
